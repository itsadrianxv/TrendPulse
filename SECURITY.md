# 安全策略

## 支持范围

当前默认只保证 `main` 分支和最新发布内容处于维护范围内。历史分支、旧快照和个人分叉不在安全修复承诺范围内。

## 报告安全问题

请不要公开提交包含漏洞细节的 Issue。

推荐顺序：

1. 优先使用 GitHub 的 Private Vulnerability Reporting（如果仓库已开启）。
2. 如果仓库暂未开启，请创建一个不包含细节的 Issue，仅说明“需要安全沟通渠道”。
3. 在提供细节时，请尽量附带影响范围、复现步骤、触发条件和建议修复方向。

## 处理预期

- 维护者会尽快确认是否收到报告。
- 若问题可以复现，会安排修复并在合适时机公开披露。
- 在修复发布前，请不要公开传播可利用细节。
